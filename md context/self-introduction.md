# 자기소개서

---

## 디자인과 코드 사이의 간극을 좁히는 퍼블리셔, 권순우입니다.

---

### 01. 퍼블리셔로서의 저

저는 Cadian(캐디안)에서 약 2년간 웹 퍼블리셔로 일하며 공식 홈페이지부터 SaaS 조직 관리 콘솔, 개발자 문서 사이트, 디자인 시스템까지 총 8개 서비스, 300건 이상의 이슈를 직접 담당했습니다.

저에게 퍼블리싱이란 디자인을 그대로 코드로 옮기는 일이 아닙니다. 디자이너의 의도를 정확히 읽고, 개발자가 유지보수하기 쉬운 구조로 번역하는 일이라고 생각합니다. 그 생각을 가장 잘 담은 작업이 바로 사내 디자인 시스템 구축이었습니다.

---

### 02. 디자인 시스템을 단독으로 설계하고 구현했습니다

재직 중 가장 도전적이었던 프로젝트는 **cadian-ui 디자인 시스템 v1.0**을 혼자 맡아 처음부터 완성한 것입니다.

여러 서비스가 각자 다른 색상 값과 컴포넌트를 사용하며 발생하는 비일관성 문제를 해결하기 위해, Figma 컴포넌트와 1:1 대응하는 디자인 토큰 체계를 먼저 설계했습니다. 그 위에 BEM 방법론 기반 SCSS 아키텍처를 적용하고, Atoms부터 Overlays · Pages까지 계층화된 컴포넌트를 구현했습니다. Storybook으로 문서화하여 팀원 누구나 컴포넌트를 확인하고 테스트할 수 있는 환경도 함께 만들었습니다.

그 과정에서 이메일·뉴스레터·이벤트 프로모션에 쓰이는 **EDM 페이지 컴포넌트**도 제작했는데, 다양한 이메일 클라이언트 환경에서의 호환성을 직접 고려한 마크업 작업은 퍼블리셔 본연의 역량을 다시 한번 확인하는 경험이었습니다.

---

### 03. SaaS 서비스의 복잡한 UI를 처음부터 만들었습니다

**organization.cadian.com**은 2년 재직 중 가장 오랜 기간 깊게 참여한 프로젝트입니다. 조직 관리 콘솔의 프론트엔드를 초기부터 구축하며, 그룹 관리·멤버 초대·SSO 인증·시리얼 관리·활동 로그·i18n까지 SaaS 서비스에서 마주치는 거의 모든 UI 패턴을 경험했습니다.

화면만 만드는 데 그치지 않았습니다. Owner / Admin / Member 역할별 권한 분기, 토큰 만료 처리, 401 인터셉터 redirect 누락, 모바일 Safari 100vh 충돌처럼 실서비스에서만 마주치는 엣지 케이스를 직접 발견하고 해결했습니다. QA 단계에서 테스트 케이스 100건 이상을 직접 수정·반영하며 코드 품질과 서비스 안정성 모두에 기여했습니다.

---

### 04. 개발 환경 표준화로 팀 전체의 생산성에 기여했습니다

**developers.cadian.com** 프로젝트에서는 ViewQ SDK의 API 예제 25종을 인터랙티브하게 구현하는 것 외에도, ESLint · Prettier 사내 표준 설정과 Git Hooks(Husky) 자동화를 도입했습니다. 퍼블리셔 한 사람의 작업이 팀 전체 코드 품질로 이어질 수 있다는 것을 그 경험에서 배웠습니다.

스크롤 위치에 따라 현재 섹션을 실시간으로 하이라이팅하는 Anchor Navigation 컴포넌트를 직접 개발한 것도 이 프로젝트에서의 기억에 남는 작업입니다. 단순 마크업을 넘어 UX까지 고민하는 습관이 이때 자연스럽게 생겼습니다.

---

### 05. 앞으로

저는 디자인과 개발 사이 어딘가에서, 두 세계가 더 잘 연결되도록 돕는 역할을 하고 싶습니다. 디자인 시스템 경험을 바탕으로 더 정교한 토큰 체계와 컴포넌트 설계를 다루고 싶고, 퍼블리셔로서 쌓아온 마크업·스타일 감각을 React 기반 프론트엔드와 더 깊이 연결하고 싶습니다.

어떤 서비스를 만나도 코드 뒤에 있는 사람의 경험을 먼저 생각하는 퍼블리셔가 되겠습니다.
